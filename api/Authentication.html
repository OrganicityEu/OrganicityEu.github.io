OrganiCity Authorisation

Version: 0.1.0-development

As users interact with the OrganiCity facility, they need to perform tasks that may not be available to any user, but only to a specific set. These are restricted by access rules, which are in turn enforced by an authorisation server.

This specification describes the endpoints that OC facility services can use to query a users information and permissions.

Schemes: http

Summary

Path Operation Description
/user/findUser GET

Returns information about a specific user

Paths

Returns information about a specific user

GET /user/findUser

This endpoint provides all known information about a specific user account. This provides more human-readable information about a user as far as it is available, which depends on the user registration source.

To find a user, this endpoints requires at least one of the possible arguments to uniquely identify the given user.
id

A valid user identifier, that denotes the user whose information is requested.

 query string
token

When a user signs into OrganiCity, she is handed a authentication token, using which the user preforms calls to other services. This token can be used here.

 query string
email

A valid email address a user signed up with, or has been supplied by its authorisation provider. As this field is not required to be provided, this might not find a user.

 query string (email)

application/json

200 OK

Information for a user with id id was found, and is returned in the body.

Body Schema: object

email: string (email)

A user's email address. This is unique among registered users.

username: string

A name this user supplied for himself. This can be used as a display name wherever this user is referenced.

404 Not Found

Not exactly one user with the given information was not found. This might mean that no or multiple users were found.
